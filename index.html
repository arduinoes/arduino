Hola y bienvenidos a este tutorial de Arduino, comenzemos por una breve explicación de Open Source Hardware, consiste en compartir el desarrollo de proyectos electrónicos de arquitectura abierta, permitiendo a otros usuarios a colaborar en el desarrollo, mejorando o añadiendo funcionalidades.



Un ejemplo son las impresoras 3D, los drones o robótica educativa donde lideran sin duda el mercado. Evolucionan rápidamente gracias a la participación de la extensas comunidades.



Disponemos de una extensa variedad de tarjetas electónicas de prototipado Open Souce. Pero sin duda el Arduino Uno sigue siendo la mejor opción para comenzar a crear los primeros proyectos, ya que tenemos una extensa documenación, así como extensiones que añadern nuevas funcionalidades.

Este es un Arduino Uno original con su protoboard.



¿Pero que és un Arduino Uno?

Bueno, la clave es su microchip ATmega 328P, un circuito integrado que funciona al igual que una computadora Dispone de varias memorias, un procesador y pines de entradas y salidas.

El diseño de la tarjeta de Arduino Uno facilita acceder a él microchip a través de un puerto USB,



poder alimentarlo con un conector de 5 a 12 voltios


y etiquetando sus pines.

SERVIDOR WEB Raspberry Pi

El Servidor en casa.

Servidor web con WordPress


Una vez que inicie la Raspberry Pi actualícela antes de instalar LAMP, utilice los siguientes comandos: 

sudo apt-get update
sudo apt-get upgrade

LAMP (Linux, Apache, MySQL y PHP)


1º Instalar Apache

Apache es un popular servidor web que se puede instalar en Raspberry Pi para alojar páginas web.

Introduzca el comando en el terminal:

sudo apt-get install apache2 -y

* ¡ATENCIÓN! Esto es una verificación, no necesita hacerlo salte al 2º paso instalar PHP


Ahora cambiamos al directorio que nos ha creado Apache. Este se localiza en: /var/www/html/index.html.

Cambiamos a este directorio y mostramos lo que se encuentra dentro:

cd /var/www/html
ls -al

Veremos varias carpetas y un archivo llamado index.html, unas columnas que nos indican los permisos de las carpetas y los archivos, el nº de archivos que contiene cada carpeta, el usuario de los directorios y archivos, el grupo al que pertenecen, el tamaño, la fecha y hora de la última modificación o creación.

Para poder editar el archivo debe modificar los permisos de usuario.

Cambie estos permisos con: 

sudo chown pi: index.html

Ahora puede editarlo con el comando:

nano index.html

Para ver el archivo en el navegador utilice la ip de la Raspberry Pi, algo como esto 192.168.x.x (cambie las "x" por las de su IP)

2º Instalar PHP

PHP es un lenguaje de programación para el desarrollo web de contenido dinámico y wordPress lo usa.

Instale PHP y su paquete con el comando:

sudo apt-get install php5 libapache2-mod-php5 -y

3º Instalar MySQL

MySQL es un sistema de gestión de bases de datos y WordPress lo utiliza.

Instale MySQL y los paquetes PHP-MySQL con los siguientes comandos:

sudo apt-get install mysql-server php5-mysql -y

Cuando instale MySQL le pedirá que introduzca una contraseña con la que mas tarde accederá a MySQL para crear una base de datos.

3º Instalar MySQL

MySQL es un sistema de gestión de bases de datos y WordPress lo utiliza.

Instale MySQL y los paquetes PHP-MySQL con los siguientes comandos:

sudo apt-get install mysql-server php5-mysql -y

Cuando instale MySQL le pedirá que introduzca una contraseña con la que mas tarde accederá a MySQL para crear una base de datos.

3º Instalar vsftpd

Para poder subir archivos a través de ftp (Filezilla) debemos de instalar vsftdp.


Instale vsftdp con el siguiente comandos:


sudo apt-get install vsftpd -y

Edite el archivo de configuración:.


sudo nano /etc/vsftpd.conf

Y modifique y descomente lo siguiente


anonymous_enable=NO
local_enable=YES
write_enable=YES


Reiniciemos el servicio.


sudo service vsftpd restart

Y cambiemos el propietario a pi.


sudo chown -R /var/www

4º Instalar WordPress

Sitúese en la carpeta /var/www/html/ cambie los privilegios de usuario "pi", borre todos los archivos y descargue el WordPress en ella. 

cd /var/www/html/
              sudo chown pi: .
              sudo rm *
              wget http://wordpress.org/latest.tar.gz

Descomprímalo, mueva el contenido a la carpeta actual "html" y elimine la carpeta "wordpress":

tar xzf latest.tar.gz
              mv wordpress/* .
              rm -rf wordpress latest.tar.gz/

Compruebe que ha descomprimido los archivos, realice un "ls":

ls

5º Crear una Base de Datos para WordPress

WordPress utiliza base de datos para su funcionamiento, créela con la contraseña utilizada en el 3º paso. 

mysql -uroot -p

Introduzca la contraseña y creé la base de datos:

mysql> create database wordpress;

Le mostrará: "Query OK, 1 row affected (0.00 sec)" para salir de "mysql>" teclee "Ctrl + D" y actualice el servidor 

sudo service apache2 restart

6º Configurar WordPress

Abra su navegador y introduzca http://suIPdirección (http://192.168.x.x).

Tendremos una página de Bienvenida a WordPress y nos invita a continuar pulsando el botón "Let's go!". Le aparecerá un formulario.

Rellene la siguientes campos:

Database Name:      wordpress
              User Name:          root
              Password:           SU CONTRASEÑA (MySQL)
              Database Host:      localhost
              Table Prefix:       wp_

Pulse el botón "Submit" Le mostrará un mensaje de error "Sorry, but I can't write the wp-config.php file", pero le da el código para hacerlo de forma manual

Vuelva a la terminal para editar el archivo

nano wp-config.php.

Copie el texto, péguelo del archivo, guárdelo con "Ctrl + X", confirme con la tecla "Y" o "S" y finalmente teclee "Enter"

Ahora puede seguir con la instalación, pulse el botón "Run the install" y le aparecerá un formulario de Bienvenida en el que introducirá el nombre de su sitio, el usuario, una contraseña segura y su correo electrónico.

Dele al botón "Install WordPress" y ya puede comenzar su "WordPress"

Introduzca sus datos para acceder al interfaz de WordPress desde cualquier ordenador con "http://tuSitio.com/wp-admin.php"

Permalinks amigables

It's recommended that you change your permalink settings to make your URLs more friendly. To do this, log in to WordPress and go to the dashboard. Go to Settings then Permalinks. Select the Post name option and click Save Changes. After saving, you will be prompted to update your .htaccess file. You probably don't have one yet, so add one in /var/www/html/ by typing nano .htaccess; note this is a hidden file, so it starts with a dot. Then paste in the contents provided:

<IfModule mod_rewrite.c>
              RewriteEngine On
              RewriteBase /
              RewriteRule ^index\.php$ - [L]
              RewriteCond %{REQUEST_FILENAME} !-f
              RewriteCond %{REQUEST_FILENAME} !-d
              RewriteRule . /index.php [L]
              </IfModule>

OWNCLOUD Raspberry pi

La nube en tu casa.


Instale "ownCloud" en Raspberry Pi


Nos dirigiremos a la página opensuse.org y elegimos Debian.


Instalación automatizada de OwnCloud

Obtengamos privilegios de super usuario:


sudo su

Añadimos la lista de los repositorios necesarios:

echo 'deb http://download.opensuse.org/repositories/isv:/ownCloud:/community/Debian_8.0/ /' >> /etc/apt/sources.list.d/owncloud.list

Actualizemos los repositorios:


apt-get update

Instalemos OwnCloud:


apt-get install owncloud

Conectar un disco duro externo para tener mas capacidad en OwnCloud

Para tener mas capacidad de almacenamiento en la Rasberry yo utilizo un disco duro externo, ya que la Raspberry utiliza una tarjeta microSD para instalar el sistema operativo.


Para conectar un disco externo en la raspberry, yo tengo un HUB de usb con alimentación propia ya que directamente en la Raspberry suele dar problemas. En la línea de comandos comenzaremos a escribir:


lsusb

Nos muestra la lista de los dispositivos usb que tenemos conectados


Bus 001 Device 007: ID 1a40:0101 Terminus Technology Inc. 4-Port HUB Bus 001 Device 006: ID 1058:0748 Western Digital Technologies, Inc. My Passport 1TB USB 3.0 Bus 001 Device 005: ID 1058:10b8 Western Digital Technologies, Inc


Con este otro comando identificamos la unidad:


sudo fdisk -l

Nos muestra algo así:


Device Boot Start End Sectors Size Id Type Units: sectors of 1 * 512 = 512 bytes Sector size (logical/physical): 512 bytes / 512 bytes Disk identifier: 0xc12da75b Device Boot Start End Sectors Size Id Type /dev/sda1 2048 4030463 4028416 1.9G 83 Linux The thumb drive is the device /dev/sda and the partition within it is /dev/sda1. We know this because fdisk reports the size as 2GB (“2063 MB”), and my thumb drive is the only 2GB drive connected to the system.


Ahora formatearemos el disco. El tipo de formato recomedado para las raspberry (Linux) es ext4.


sudo mkfs -t ext4 /dev/sda1

*Si te dice que está montado lo desmontaremos:


sudo umount /dev/sda1

Ya formateado creo un directorio:


sudo mkdir /disk

Ahora monto el disco externo al directorio:


sudo mount /dev/sda1 /disk

Lo verificamos.


df

Veremos algo así.

/dev/sda1 1949776 2952 1829732 1% /disk


Creemos ahora una carpeta para Owncloud.


sudo mkdir /disk/owncloud

Y le otrogamos privilegios de propietario:


sudo chown -R www-data:www-data /disk/owncloud

Podremos ahora con la "nuestraIP/owncloud" iniciar nuestra nube. Introducimos un nombre de usuario y una contraseña y modificamos el directorio data a "disk/owncloud" que hemos creado.

Otra alternativa de inicio.

Iniciamos la carpeta por defecto de owncloud, para ello debemos asignarle primero los privilegios de propietario:


sudo chown -R www-data:www-data var/www/owncloud

Una vez iniciado owncloud realizamos el cambio de directorio.


Cambiemos a la carpeta de "owncloud" donde se encuentra la carpeta "data":


cd /var/www/html/owncloud

Y luego copiamos "data" a la carpeta del del disco


sudo cp -rp data /disk/owncloud/data

Testeamos la copia de los datos.


sudo ls -l /disk/owncloud/data

Mostrará algo como esto.


-rw-r--r-- 1 www-data www-data 0 Feb 16 11:23 index.html drwxr-xr-x 4 www-data www-data 4096 Feb 16 11:27 fred drwxr-xr-x 2 www-data www-data 4096 Feb 16 11:23 updater_backup


Ahora necesitamos reconfigurar owncloud editando "config.php"


sudo nano /var/www/owncloud/config/config.php

Y Cambiamos:


'datadirectory' => '/var/www/owncloud/data'
por:
'datadirectory' => '/disk/owncloud/data'


Y salvamos el archivo.


Tenemos que cambiar los privilegios del propietario del disco.


sudo chown -R www-data:www-data /disk/owncloud/data

Si deseamos acceder a la carpeta data debemos modificar sus privilegios


sudo chmod 777 -R /disk/owncloud/data

Modificar los límites de subida y bajada

Editar el archivo /etc/php5/apache2/php.ini


sudo nano /etc/php5/apache2/php.ini

Cambia estas dos líneas:


post_max_size = 8M upload_max_filesize = 2M por: post_max_size = 2000M upload_max_filesize = 2000M


Cambiar el límite de owncloud


sudo nano /var/www/owncloud/.htaccess

upload_max_filesize por: limited 2000M


Listo!

Que és GRBL


Es una library (bliblioteca) de arduino que nos facilita el control de los motores, servos ... a través de una aplicación "Universal Gcode Sender" podemos configurar/personalizar nuestra máquina de forma sencilla a través de unos comandos y mandarle nuertro archivo Gcode para que corte/grabe/dibuje nuestro diseño

$$

Mostrará la configuración por defecto:

Acceda desde la terminal (pestaña de comandos) de Universal Gcode Sender, la configuración de su firmware escribiendo $$.

Compruebe los parámetros: 

$ 100

                                  $ 101

Definen cuántos pasos se requieren para que la máquina avance 1 mm. Teniendo en cuenta que utiliza un motor de 200 pasos, una polea de 20 dientes y la correa GT2 (paso de 2 mm), los valores correctos para ambos parámetros serían 80.

Si no están con estos valores, escriba: 
$100

                                  $101

"$ 100 = 80 + Enter" en el terminal para ajustar el eje X. Escriba “$ 101 = 80 + Enter” para ajustar el eje Y. Listo, su máquina ahora dibujará exactamente las mismas dimensiones que su dibujo de Inkscape.

El firmware que tenemos que subir a Arduino es una versión del Grbl modificada para poder trabajar con el servo, que subierá y bajará el lápiz cuando toque grbl 0.9i with Servo motor support. A parte tendremos que hacer un pequeño cambio en el fichero config.h para que esté configurado como una máquina CoreXY, descomenta define COREXY // Default disabled. Uncomment to enable..

  1. Configuremos los pasos por milímetro
  2. Comprobemos que los ejes no esten invertidos (efecto espejo)
  3. Configuremos la velocidad
  4. Configuremos el spinle
  5. Volvamos a refrescar

Robot educativo para primaria.

Se trata de un robot educativo para niños de primaria el cual demuestra que el robot obedece nuestras instrucciones a través de comandos simples como avanaza, retrocede, gira a la derecha / izquierda.
