ESP8266



Referencias Arduino


WiFi.begin()

Inicializa la configuración de red de la biblioteca WiFi y proporciona el estado actual.

WiFi.status()

Devuelve el estado de la conexión.

WiFi.localIP()

Obtiene la dirección IP del dispositivo WiFi

WiFi.begin(ssid, password);

while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
}

Serial.println("");
Serial.println("WiFi conectado");
Serial.println("Dirección IP: ");
Serial.println(WiFi.localIP());
}

WiFiClient()
WiFiClient client;

Crea un cliente que puede conectarse a una dirección IP y puerto de Internet especificados como se define en client.connect ()

cliente.connect()

Se conecta a la dirección IP y al puerto especificados en el constructor. El valor de retorno indica éxito o fracaso. connect () también admite búsquedas DNS cuando se usa un nombre de dominio (por ejemplo, google.com).

 if (client.connect(servername, 80)) {
  Serial.println("connected");
  // Make a HTTP request:
  client.println("GET /search?q=arduino HTTP/1.0");
  client.println();
}

client.available()

Devuelve el número de bytes disponibles para leer (es decir, la cantidad de datos que el servidor al que está conectado ha escrito al cliente).

client.read()

Devuelve el número de bytes disponibles para leer (es decir, la cantidad de datos que el servidor al que está conectado ha escrito al cliente).

  // Si hay bytes entrantes disponibles
// desde el servidor, léelos e imprímelos read:
if (client.available()) {
char c = client.read();
Serial.print(c);
}

cliente.println()

Envía datos, seguidos de un retorno de carro y una nueva línea, al servidor al que está conectado un cliente. Imprime los números como una secuencia de dígitos, cada uno un carácter ASCII (por ejemplo, el número 123 se envía como los tres caracteres '1', '2', '3').

// Esto envia strings "cadenas" al servidor
  Serial.println("Enviando datos al servidor");
  if (client.connected()) {
    client.println("Hola desde ESP8266");
  }

WiFiServer server(80);

Crea un servidor que escucha las conexiones entrantes en el puerto especificado.

server.begin()

Le dice al servidor que comience a escuchar las conexiones entrantes

server.setNoDelay(true);

Este algoritmo está destinado a reducir el tráfico TCP / IP de pequeños paquetes enviados a través de la red combinando una cantidad de pequeños mensajes salientes y enviándolos todos a la vez. La desventaja de este enfoque es retrasar efectivamente los mensajes individuales hasta que se reúna un paquete lo suficientemente grande.
